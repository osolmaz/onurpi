# Pi Must Win

- Keep this package focused on truthful, recognizable Pi branding and attribution.
- Add independent branding capabilities under `features/` and register them from `index.ts`.
- Do not persist Git hooks or repository configuration.
- Preserve existing repository hooks and propagate their failures.
- Sanitize external values before writing them to commit messages or shell environments.
- Update `UPSTREAM.md` and retain upstream license notices when adopting upstream changes.
- Run `npm run check`, `npm run mutate`, and `npm run slophammer` before finishing.
